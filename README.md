# Cybersecurity Incident Response Framework
> Academic Project — 4-Week Plan

## Objective
Build a complete cybersecurity incident response framework covering planning, execution, and review based on core cybersecurity principles.

---

## Team Roles

| Role | Responsibility |
|------|---------------|
| 🔵 Role 1 — Research Lead | Finds sources, researches concepts, writes executive summary |
| 🟢 Role 2 — Technical Lead | Architecture, network discovery, simulation execution |
| 🟡 Role 3 — Documentation Lead | Writes and compiles all formal reports |
| 🟣 Role 4 — Presenter Lead | Builds all slides and leads the presentation |

2. Role 2 delivers architecture diagram by Week 2 Day 3–4

4. No deliverable is submitted without full team review

### Week 1
- Report on cybersecurity concepts and network discovery techniques
- Presentation slides summarizing key findings

### Week 2
- Detailed incident response plan document
- Secure architecture and system hardening report

### Week 3
- Incident simulation report
- Response documentation (containment, eradication, recovery)

### Week 4
